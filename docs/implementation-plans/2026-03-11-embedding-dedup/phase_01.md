# Embedding Dedup Implementation Plan

**Goal:** Add embedding column and duplicate status to the articles table schema

**Architecture:** Extend the existing Drizzle ORM schema with a nullable JSON embedding column and a new `pending_dedup` / `duplicate` status value, then generate and apply the migration.

**Tech Stack:** Drizzle ORM, better-sqlite3, SQLite

**Scope:** 6 phases from original design (phases 1-6)

**Codebase verified:** 2026-03-11

---

## Acceptance Criteria Coverage

This phase implements infrastructure only. No acceptance criteria are directly tested here — the schema changes enable all subsequent phases.

**Verifies: None** (infrastructure phase, verified operationally)

### Task 1: Extend articles status enum and add embedding column

**Files:**
- Modify: `src/db/schema.ts:43-44` (status enum), add embedding column after line 42

**Step 1: Update the status enum to include new values**

In `src/db/schema.ts`, change the `status` field's enum array from:

```typescript
status: text("status", {
  enum: ["pending_assessment", "assessed", "failed"],
})
```

to:

```typescript
status: text("status", {
  enum: ["pending_dedup", "pending_assessment", "assessed", "duplicate", "failed"],
})
```

This adds `pending_dedup` (initial status for new articles entering the embedding dedup pipeline) and `duplicate` (articles identified as duplicates by cosine similarity).

Also change the `.default("pending_assessment")` on line 47 to `.default("pending_dedup")`:

```typescript
.default("pending_dedup"),
```

This ensures any code path that inserts an article without explicitly specifying a status will enter the embedding dedup pipeline rather than bypassing it.

**Step 2: Add the embedding column**

Add a new column after the `metadata` column (line 42) in the articles table definition:

```typescript
embedding: text("embedding", { mode: "json" }).$type<ReadonlyArray<number>>(),
```

This follows the existing JSON column pattern used by `metadata` and `extractorConfig`. The column is nullable (no `.notNull()`) since articles may not have embeddings (Ollama unavailable, or legacy articles).

**Step 3: Verify the file compiles**

Run: `npx tsc --noEmit`
Expected: No type errors

**Step 4: Commit**

```bash
git add src/db/schema.ts
git commit -m "feat(db): add embedding column and dedup statuses to articles schema"
```

### Task 2: Generate and verify Drizzle migration

**Files:**
- Generated: `drizzle/` (new migration file, exact name generated by drizzle-kit)

**Step 1: Generate the migration**

Run: `npm run db:generate`
Expected: Drizzle-kit detects the schema change and generates a new migration SQL file in `drizzle/`. The output should mention adding the `embedding` column and modifying the `status` column.

**Step 2: Inspect the generated migration**

Open the newly generated migration SQL file in `drizzle/` and verify it contains:
- An `ALTER TABLE articles ADD COLUMN embedding text;` statement (or equivalent)

Note: SQLite does not support altering enum constraints — the status enum is enforced at the Drizzle ORM level, not the database level. The migration should only need to add the new column.

**Step 3: Apply the migration to verify it works**

Run: `npm run db:push`
Expected: Schema pushed successfully without errors

**Step 4: Run existing tests to verify no regressions**

Run: `npm test`
Expected: All existing tests pass. The in-memory test database created by `createTestDatabase()` applies migrations, so it will pick up the new column automatically.

**Step 5: Commit**

```bash
git add drizzle/
git commit -m "chore(db): generate migration for embedding column and dedup statuses"
```
